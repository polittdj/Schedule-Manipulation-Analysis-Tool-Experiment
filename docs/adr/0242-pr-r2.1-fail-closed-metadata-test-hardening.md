# ADR-0242 — PR-R2.1: fail-closed on missing dist metadata + startup-guard test hardening + doc-census fix

## Status

Accepted. Lead-validated follow-ups to PR-R2 (ADR-0241), surfaced by an adversarial
multi-agent re-review of that diff (3 reviewer lenses × per-finding skeptics). Every finding
below was re-verified against the code before it was actioned (CLAUDE.md model/audit protocol);
findings that failed re-verification were dropped.

## Context

PR-R2 wired the two dead Law-1 defenses (`configure_logging` + `assert_local_only`) into every
startup path and enumerated the air-gap scan. The re-review found the wiring correct but exposed
three real gaps — one a self-inflicted robustness regression, the others tests that could pass
vacuously and so would not actually pin what they claim to.

- **Regression (net_guard):** `assert_local_only()` → `runtime_requirement_names()` calls
  `importlib.metadata.requires(DIST_NAME)`, which raises `PackageNotFoundError` when the dist
  metadata is absent. That is unreachable on the shipped pip-installed wheel, but reachable from a
  raw source checkout run without `pip install`, or a frozen build that didn't bundle metadata —
  and PR-R2 moved this call onto every process start (launcher, `create_app`, the report CLI). The
  bare exception would crash the entry point opaquely, and silently under the `pythonw` desktop
  icon (stderr already rebound to devnull). `web/app.py` already anticipates this run mode and
  guards the twin `importlib.metadata.version()` lookup — the guard three lines into `create_app`
  did not.
- **Vacuous startup-guard tests:** `configure_logging` installs a process-global handler and is
  idempotent-by-replacement, so once any test runs, the handler lingers. The
  "handler present after calling my entry point" assertions in `test_startup_guards`,
  `test_launcher`, and `test_cli_guards` then pass off that leftover — they'd still pass if the
  entry point's `configure_logging()` call were deleted. The launcher's OWN call was doubly
  masked: `main()` builds the app via `create_app()`, which installs the handler regardless.
- **Air-gap pin not bound to the walk:** `_MUST_ENUMERATE` was asserted against the raw
  `app.routes` table, not the set `_get_route_paths()` actually walks, so a regression confined to
  the walk (a broken GET filter, an added skip) left the pin green while the scan went vacuous.
  The vendored-asset walk was non-recursive (a future `static/vendor/x.js` would be served but
  never scanned), and the loopback allowlist was a whole-URL substring test
  (`http://127.0.0.1.evil.com/…` would be exempted).
- **Doc census drift:** `REPO-INVENTORY.md` still claimed 66 static assets (tree has 58) and stale
  test-file counts that PR-R2's new files widened.

## Decision

1. **Fail closed, legibly (net_guard):** `runtime_requirement_names()` catches
   `PackageNotFoundError` and re-raises `CUIEgressError` with a self-explaining message ("cannot
   verify the runtime dependency closure — package metadata … missing; install the package, or
   bundle its metadata in a frozen build"). A CUI tool that cannot verify its own dependency
   closure must refuse to start, not run unverified (Law 1) — but with a clear cause, not an opaque
   `PackageNotFoundError`. The shipped wheel always carries metadata, so this never fires for a
   real install. New `tests/guards/test_egress.py` case pins it.
2. **Reset fixture kills the vacuous pass:** a shared `reset_redacting_logging` fixture
   (`tests/conftest.py`) clears the `schedule_forensics` logger handlers + `propagate` + the
   module `_configured` flag before each startup-guard test and restores them after, so each test
   proves ITS entry point freshly installs the handler. A new `test_launcher` case records the
   launcher's OWN `configure_logging`/`assert_local_only` calls at the launcher namespace and
   asserts both run BEFORE `create_app()` — pinning the launcher's wiring independently of the app.
   Both were mutation-verified: deleting either entry point's `configure_logging()` now fails a
   test (it did not before).
3. **Air-gap pin binds the walk:** `_get_route_paths()` returns the route pattern too, and
   `_MUST_ENUMERATE` is asserted against the WALK OUTPUT (mutation-verified: a lowercased GET
   filter now fails the pin). The vendored-asset walk uses `rglob` (subdirectories included), and
   the loopback exemption parses the URL host and matches it exactly instead of substring-testing
   the whole URL.
4. **Census fixed:** `REPO-INVENTORY.md` static assets 66 → 58 (54 JS + 4 CSS); `tests/web`
   102 → 108, `tests/exhibits` 2 → 3, each noting the new Law-1 startup-guard files.

## Consequences

- The Law-1 startup wiring is now genuinely pinned: the tests fail if the wiring regresses, which
  the PR-R2 versions did not guarantee. This is the "make the guard's test real" posture PR-R2 was
  about, applied to PR-R2's own tests.
- `assert_local_only()` stays fail-closed but is now diagnosable in the raw-source / frozen-build
  edge; a future PyInstaller build must bundle `schedule-forensics` metadata (or it will refuse to
  start with the clear message — by design).
- No engine, metric, parity, or UI surface touched; the Gantt governing-calendar shading change
  (operator-approved Option B) ships separately.
